Custom data¶

First, we will consider a toy example to understand how PDP and ALE, two global interpretation methods, handle correlated features. Imagine a simple data generating process (DGP) with two features, $x_1$ and $x_2$, that are highly correlated.

Furthermore, we assume that the outcome $y$ is created via the following DGP: $y = x_1 + x_2^2 + \epsilon, \ \epsilon \sim \mathcal{N}(0, 0.05^2)$.

def compute_pdp(model, X, feature, grid_size=30):
    """Compute Partial Dependence Plot for a feature"""
    # Get feature name and values
    feature_name = X.columns[feature]
    feature_values = X[feature_name].values
    
    # Create grid for the feature
    grid_min, grid_max = feature_values.min(), feature_values.max()
    grid_points = np.linspace(grid_min, grid_max, grid_size)
    
    # Initialize array to store average predictions
    pdp_values = np.zeros(len(grid_points))
    
    # For each grid point
    for i, grid_val in enumerate(grid_points):
        # Create a copy of the dataset
        X_modified = X.copy()
        # Set the feature value to the grid point for all instances
        X_modified[feature_name] = grid_val
        # Predict with the modified dataset
        predictions = model.predict(X_modified)
        # Store the average prediction
        pdp_values[i] = np.mean(predictions)
    
    return grid_points, pdp_values

def compute_ale(model, X, feature, grid_size=30):
    """Compute Accumulated Local Effects for a feature"""
    # Get feature name and values
    feature_name = X.columns[feature]
    feature_values = X[feature_name].values
    
    # Create quantile-based grid for the feature
    quantiles = np.linspace(0, 1, grid_size + 1)
    grid_points = np.quantile(feature_values, quantiles)
    
    # Ensure grid points are unique
    grid_points = np.unique(grid_points)
    
    # Initialize arrays to store effects
    ale_values = np.zeros(len(grid_points) - 1)
    
    # For each interval in the grid
    for i in range(len(grid_points) - 1):
        # Find instances that fall within this interval
        mask = (feature_values >= grid_points[i]) & (feature_values <= grid_points[i + 1])
        instances = X[mask].copy()
        
        if len(instances) > 0:
            # Create two modified datasets with feature values at lower and upper bounds
            X_lower = instances.copy()
            X_upper = instances.copy()
            X_lower[feature_name] = grid_points[i]
            X_upper[feature_name] = grid_points[i + 1]
            
            # Predict with both modified datasets
            y_lower = model.predict(X_lower)
            y_upper = model.predict(X_upper)
            
            # Calculate local effect for each instance
            local_effects = y_upper - y_lower
            
            # Average local effects for this interval
            ale_values[i] = np.mean(local_effects)
    
    # Accumulate effects
    ale_accumulated = np.cumsum(ale_values)
    
    # Center ALE to have mean effect of 0
    ale_accumulated = ale_accumulated - np.mean(ale_accumulated)
    
    # Return grid midpoints and ALE values
    grid_midpoints = (grid_points[:-1] + grid_points[1:]) / 2
    return grid_midpoints, ale_accumulated

# ALE for x1 on test data (feature index 0)

Exercise 4: PFI¶

Feature permutation is the process of randomly shuffling the observed values of a single feature in the dataset. This breaks the feature's relationship with the target and all other features, while preserving its marginal distribution.

The PFI measures the change in model performance before and after permuting a feature. Features whose permutation leads to a large decrease in performance are considered more important.

In scikit-learn, we can compute PFI using the permutation_importance() function.
We will evaluate the Random Forest model you fitted in the previous exercise, using the MSE as the performance metric.

  3. Plot the feature importances as a horizontal bar plot, sorted by importance.

Exercise 5: LOCO¶

The LOCO method measures feature importance by removing each feature one at a time, retraining the model, and comparing its performance to the baseline model trained on all features. The performance drop indicates the feature's importance.

In this exercise, you will implement a function that:

  1. Compute baseline performance using cross_val_score().
  2. For each feature:
    • Drop it from X.
    • Refit the model on the reduced dataset.
    • Compute the performance drop compared to the baseline.
  3. Return the LOCO values for all features.
Hint 1: Use `clone()` to reset the model for each reduced dataset and use `cross_val_score()` with `scoring="neg_mean_squared_error"` to evaluate model performance.
Hint 2: The function should have the following signature and docstring:

    """

from sklearn.model_selection import cross_val_score, KFold, BaseCrossValidator
from sklearn.base import clone, BaseEstimator


def loco(X: pd.DataFrame, 
         y: pd.Series, 
         learner: BaseEstimator, 
         cv: BaseCrossValidator, 
         scoring: str) -> pd.Series:
    """
    Compute LOCO (Leave-One-Covariate-Out) values.
    
    Parameters:
      X: Feature data.
      y: Target variable.
      learner: The machine learning model to evaluate.
      cv: cross-validation splitter e.g., KFold, StratifiedKFold.
      scoring : Scoring metric; for MSE, use 'neg_mean_squared_error'.
    
    Returns:
      pd.Series
        A series with LOCO values for each feature.
    """
    # Compute baseline performance on the full dataset:
    baseline_scores = cross_val_score(learner, X, y, cv=cv, scoring=scoring)
    # Aggregate baseline performance (mean across folds)
    baseline_perf = -np.mean(baseline_scores)
    
    loco_values = {}
    for feature in X.columns:
        # Create a reduced dataset by dropping one feature:
        X_reduced = X.drop(columns=[feature])
        
        # Clone the learner to ensure independence for each iteration:
        learner_reduced = clone(learner)
        
        # Compute performance on the reduced dataset:
        reduced_scores = cross_val_score(learner_reduced, X_reduced, y, cv=cv, scoring=scoring, n_jobs=-1)
        reduced_perf = -np.mean(reduced_scores)
        
        # LOCO: the increase in loss compared to the full model
        loco_values[feature] = reduced_perf - baseline_perf
    
    return pd.Series(loco_values)
