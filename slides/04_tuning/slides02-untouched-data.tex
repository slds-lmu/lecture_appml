\begin{frame}{Overtuning}
\small
Let $\lams_t$ be the incumbent (best-performing configuration) at step $t$, i.e. the configuration with the best validation score seen so far.

\vspace{0.5em}

\textbf{Meta-overfitting:} validation looks better than test for the current incumbent
$$
\mathrm{of}_t = \widehat{\mathrm{test}}(\lams_t) - \widehat{\mathrm{val}}(\lams_t)
$$

\textbf{Overtuning:} the current incumbent is worse on test than the best incumbent seen so far
$$
\mathrm{ot}_t = \widehat{\mathrm{test}}(\lams_t) - \min_{t^\prime \le t} \widehat{\mathrm{test}}(\lams_{t^\prime})
$$

\begin{itemize}
    \item Meta-overfitting is a \textbf{gap} between validation and test.
    \item Overtuning is a \textbf{selection error}: we keep improving validation but select a worse model.
    \item Nonzero meta-overfitting is possible without overtuning.
    \item In HPO, overtuning is usually the more important practical problem.
\end{itemize}

\end{frame}

\begin{frame}{Remedies against Overtuning}
    \begin{itemize}
        \item \textbf{More data / larger validation sets:} reduces noise in validation estimates.
        \item \textbf{Repeated CV or reshuffling splits:} stabilizes the HPO objective.
        \item \textbf{Be skeptical about tiny late gains:} they may be pure validation noise.
        \item \textbf{Keep strong baselines:} defaults or early incumbents can already generalize best.
    \end{itemize}

\vspace{0.5em}

Main message from the paper: overtuning is especially relevant in the \textbf{small-data regime}.

\end{frame}

\section{Recap}

\begin{frame}{Importance of CV on Kaggle}

\begin{itemize}
\setlength\itemsep{1em}
    \item Understand the data - After you download the data, start exploring features. Look at data types. Check variable classes. Create some univariate - bivariate plots to understand the nature of variables.
    \item Understand the metric to optimize - Every problem comes with a unique evaluation metric. It's imperative for you to understand it, specially how does it change with target variable.
    \item Decide cross validation strategy - To avoid overfitting, make sure you've set up a cross validation strategy in early stages. \textbf{A nice CV strategy will help you get reliable score on leaderboard}.
    \item Start hyper parameter tuning
\end{itemize}

Source: KazAnova, former Kaggle Champion at \href{https://www.hackerearth.com/practice/machine-learning/advanced-techniques/winning-tips-machine-learning-competitions-kazanova-current-kaggle-3/tutorial/}{hackerearth.com tutorial}
\end{frame}

        \item Use advanced and robust HPO
        \item Include your preprocessing in the splitting
