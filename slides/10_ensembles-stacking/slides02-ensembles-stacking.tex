\begin{vbframe}{From Averaging to Stacking}
\vfill
So far: weighted model averaging uses a \textbf{fixed} aggregation function $G$:
$$f_{\mathbf{w}}(\xv) = \sum_{m=1}^{M} w_m \cdot \bl[m](\xv)$$
\vfill
Key limitation: weights $w_m$ are \textbf{global} --- every input $\xv$ gets the same weighting of models.
\vfill
What if model 1 is better in one region of the input space, and model 2 in another?

$\rightarrow$ \textbf{Idea:} Replace the fixed $G$ with a \textbf{learned model} that can adapt the combination to the input.
This is \textbf{stacking}.
\vfill
\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Stacking}
Also called stacked generalization (\furtherreading{WOLPERT1992}): train multiple base models (Level 0) on the same data and use their predictions as input features for a Level 1 model that makes the final ensemble prediction $\rightarrow$ heterogeneous ensembling
\vspace{1em}

    \begin{column}{0.5\textwidth}
        \vspace{-1em}

    \begin{column}{0.5\textwidth}
        Steps:
        \begin{enumerate}
            \item Base model training (Level 0)
            \item Collect predictions
            \item Level 1 model training
        \end{enumerate}
    \end{column}
\end{columns}
\vspace{1em}

In contrast to model averaging, stacking already combines predictions during training.

\end{vbframe}

\begin{vbframe}{Stacking}
\footnotesize
\textbf{Algorithm:} Single Layer Stacking
\begin{algorithmic}
    \Require Training data $\D = \Dset$
    \Ensure Stacked Ensemble $f^{[M]}$

    \State
    \State \textbf{Step 1: Learn Level 0 base models}
    \For{$m \leftarrow 1$ to $M$}
        \State Fit base model $\bl[m]$ on $\D$
    \EndFor

    \State
    \State \textbf{Step 2: Construct new dataset $\D^\prime$ from $\D$}
    \For{$i \leftarrow 1$ to $n$}
        \State Add $(\xv^{\prime(i)}, y^{(i)})$ to new dataset, where $\xv^{\prime(i)} = (\bl[1](\xv^{(i)}), \ldots, \bl[M](\xv^{(i)}))$
    \EndFor

    \State
    \State \textbf{Step 3: Fit Level 1 model $f^{[M]}$ on $\D^\prime$}

    \State \Return $f^{[M]}$, $\bl[1], \ldots, \bl[M]$
\end{algorithmic}
NB: We can either drop the original features or keep them when learning the Level 1 model.

\end{vbframe}

\footnotesize

\vspace{0.5em}

\textbf{Obs} & \textbf{RF} $\hat{\pi}^{[1]}$ & \textbf{k-NN} $\hat{\pi}^{[2]}$ & \textbf{LogReg} $\hat{\pi}^{[3]}$ & $y$ \\
\hline
1  & 0.82 & 0.91 & 0.65 & 1 \\
2  & 0.35 & 0.28 & 0.41 & 0 \\

    \item The meta-model can learn, e.g., ``trust RF when it is confident, but rely on k-NN otherwise''
\end{itemize}

\begin{vbframe}{Why Not Use Training Predictions Directly?}
\vfill
\textbf{Problem:} If base models predict on the same data they trained on, the predictions can be overly optimistic.
\vfill
\textbf{Example:} A 1-nearest-neighbor classifier achieves 0\% error on its training data, but may have 30\% error on unseen data.
$\rightarrow$ The meta-model would see 0\% error and learn to fully trust this model.
\vfill
\textbf{Solution:} Use \textbf{cross-validated (out-of-fold) predictions} to construct the meta-feature matrix.
Each base model predicts only on data it was \emph{not} trained on $\rightarrow$ predictions reflect true generalization quality.
\vfill

\end{vbframe}

        \end{figure}

    \begin{column}{0.4\textwidth}
        \footnotesize
        Why use cross-validated predictions?
        \begin{itemize}
            \item \textbf{Avoids information leakage:} without CV, base models predict on their own training data $\rightarrow$ overconfident $\rightarrow$ meta-model overfits
            \item \textbf{Realistic error signal:} OOF predictions reflect true model quality
            \item \textbf{Data-efficient:} all $n$ observations get a prediction $\rightarrow$ full-size meta-dataset
        \end{itemize}
        \vspace{0.5em}
        \textbf{Blending} vs.\ \textbf{Stacking:}
        \begin{itemize}
            \item Blending: single hold-out split (simpler, wastes data)
            \item Stacking: $K$-fold CV (data-efficient, more complex)
        \end{itemize}
    \end{column}
\end{columns}
\end{vbframe}

\begin{vbframe}{Stacking Cross-validated}
\footnotesize
\textbf{Algorithm:} Single Layer Stacking with Cross-Validation
\begin{algorithmic}
    \Require Training data $\D = \Dset$, number of folds $K$
    \Ensure Stacked Ensemble $f^{[M]}$

    \For{$k \leftarrow 1$ to $K$}
        \State \textbf{Step 1: Learn fold-specific Level 0 base models}
        \For{$m \leftarrow 1$ to $M$}
            \State Fit base model $b_k^{[m]}$ on $\D_{\mathrm{train}}^{(k)}$
        \EndFor
        \State
        \State \textbf{Step 2: Add out-of-fold predictions to new dataset $\D^\prime$}
        \ForAll{$(\xv^{(i)}, y^{(i)}) \in \D_{\mathrm{test}}^{(k)}$}
            \State Add $(\xv^{\prime(i)}, y^{(i)})$ to new dataset, where $\xv^{\prime(i)} = (b_k^{[1]}(\xv^{(i)}), \ldots, b_k^{[M]}(\xv^{(i)}))$
        \EndFor
    \EndFor

    \State
    \State \textbf{Step 3: Fit Level 1 model $f^{[M]}$ on $\D^\prime$}

    \State \Return $f^{[M]}$
\end{algorithmic}
NB: For prediction, the base models must be retrained on the full dataset $\D$ (see Forward Pass).

\end{vbframe}

\begin{vbframe}{Higher Level Models: Meta-Learner Choice}
\vfill
Popular choices of Level 1 or Level 2 models:
\begin{itemize}
    \item \textbf{Linear / logistic regression} (most common)
    \begin{itemize}
        \item Simple meta-model reduces overfitting risk on the small meta-feature space ($M$ columns)
        \item Non-negative weight constraints can improve stability
    \end{itemize}
    \item \textbf{GES:} greedily selects and averages predictions of the models at the previous level
    %\item random forest (especially in the case of multilevel stacking)
\end{itemize}
\vfill
NB: A powerful meta-model (e.g., gradient boosting) can overfit the meta-features, especially with few base models. Simpler is often better here.
\vfill
\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Higher Level Models: Meta-Feature Design}
\vfill
\textbf{Probabilities vs.\ hard labels} as meta-features (\furtherreading{TINGWITTEN1999}):
\begin{itemize}
    \item Hard labels destroy confidence information: predictions 0.51 and 0.99 both map to class 1, but mean very different things
    \item Probability predictions preserve this signal $\rightarrow$ meta-model can learn \emph{when} a base model is confident
    \item $\Rightarrow$ Always prefer probability outputs as meta-features for classification
\end{itemize}
\vfill
\textbf{Include original features $\Xmat$} alongside meta-features?
\begin{itemize}
    \item Can help when base models miss informative features (e.g., a feature no base model uses)
    \item Increases overfitting risk: the meta-model now has $p + M$ features instead of $M$
    \item In practice: try both, evaluate via nested CV
\end{itemize}
\vfill
\end{vbframe}

\begin{vbframe}{Forward Pass / Prediction}
%\footnotesize
\textbf{Assume:} trained base models $\bl[1], \dots, \bl[M]$ and meta-model $f^{[M]}$

\begin{algorithmic}
    \Require Test data $\Xmat_{\text{test}} = (\xv^{(1)}, \dots, \xv^{(n_{\text{test}})})$
    \Ensure Predictions $\hat{\yv}_{\text{test}}$

    \State \textbf{Step 1: Predict with each base model}
    \For{$m \leftarrow 1$ to $M$}
        \State $\hat{\yv}^{[m]} \leftarrow \bl[m](\Xmat_{\text{test}})$
    \EndFor

    \State
    \State \textbf{Step 2: Construct meta-feature matrix}
    \State $\Xmat^\prime \leftarrow [\hat{\yv}^{[1]}, \dots, \hat{\yv}^{[M]}]$ or $\Xmat^\prime \leftarrow [\Xmat_{\text{test}}, \hat{\yv}^{[1]}, \dots, \hat{\yv}^{[M]}]$ (if original features are retained)

    \State
    \State \textbf{Step 3: Predict with the meta-model}
    \State \Return $f^{[M]}(\Xmat^\prime)$
\end{algorithmic}
\end{vbframe}

\begin{vbframe}{Conceptual Check}

\vfill
What other methods do the following level-1 classifiers recover?
\vfill
\begin{itemize}
    \item Fit the level-1 training data with a linear model that only depends on one feature, and that feature is chosen such that $\rho$, our metrics of interest, is optimized.
    \item The level-1 model is a linear model that assigns equal weight $w_{m} = \frac{1}{M}$ to every input feature.
\end{itemize}
\vfill

\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Conceptual Check -- Answers}

\vfill
\begin{itemize}
    \item A linear model that selects the single best input feature (= base model prediction) and optimizes $\rho$ is equivalent to \textbf{model selection}: simply pick the best-performing base model.
    \item A linear model with equal weights $w_{m} = \frac{1}{M}$ for every input feature is equivalent to \textbf{simple model averaging}: $\frac{1}{M} \sum_{m=1}^{M} \bl[m](\xv)$.
\end{itemize}
\vfill
$\rightarrow$ Stacking with a flexible level-1 model generalizes both model selection and model averaging!
\vfill

\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{When Does Stacking Help?}
\vfill
Stacking tends to \textbf{help} when:
\begin{itemize}
    \item Base models are diverse (different algorithms, different hyperparameters)
    \item Base models have complementary strengths (good in different regions of input space)
    \item Training set is large enough for reliable out-of-fold predictions
\end{itemize}
\vfill
Stacking can \textbf{hurt} when:
\begin{itemize}
    \item Base models are too similar $\rightarrow$ meta-model has nothing useful to learn
    \item Dataset is small $\rightarrow$ out-of-fold predictions are noisy, meta-model overfits
    \item Only one base model clearly dominates $\rightarrow$ model selection is simpler and equally effective
\end{itemize}
\vfill
NB: Simple model averaging is a surprisingly strong baseline; stacking is not always worth the added complexity.
\vfill
\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Practical Tips for Stacking}
\vfill
\begin{itemize}
    \item \textbf{Number of folds:} $K = 5$ or $K = 10$ is typical for CV stacking. Smaller $K$ $\rightarrow$ more bias in OOF predictions; larger $K$ $\rightarrow$ more computation.
    \item \textbf{Number of base models:} Diminishing returns after $\sim$5--10 diverse models. Focus on diversity over quantity.
    \item \textbf{Tune first, then stack:} Treat hyperparameter tuning of base models and stacking as two separate stages. Do not tune base models \emph{inside} the stacking CV loop.
    \item \textbf{Avoid double-dipping:} The OOF predictions used to train the meta-model should \emph{not} also be used to select which base models to include $\rightarrow$ use nested CV for an honest estimate of the stacked ensemble's performance.
    \item \textbf{Start simple:} Try equal-weight averaging first. Only move to stacking if averaging leaves clear room for improvement.
\end{itemize}
\vfill
\end{vbframe}

\begin{vbframe}{Summary}
\vfill
\begin{itemize}
    \item \textbf{Stacking} replaces a fixed aggregation function with a \textbf{learned meta-model} trained on base model predictions
    \item Use \textbf{cross-validated (out-of-fold) predictions} to avoid information leakage when constructing the meta-feature matrix
    \item After CV stacking, \textbf{retrain} base models on full data for deployment
    \item Keep the meta-model \textbf{simple} (e.g., linear/logistic regression) and use \textbf{probability} outputs as meta-features
    \item Stacking generalizes both model selection and model averaging
    \item Multi-layer stacking (e.g., AutoGluon) extends this to multiple levels with repeated bagging and feature passthrough
\end{itemize}
\vfill
\end{vbframe}
