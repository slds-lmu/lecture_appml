\begin{vbframe}{Ensembling in Machine Learning}
\begin{itemize}
\item Techniques that combine predictions of multiple base models
\item Goal: Improve predictive performance and model robustness
\item Idea: Errors of a single model will likely be compensated by other models, and as a result, the overall prediction performance of the ensemble would be better than that of a single model
\end{itemize}

\begin{vbframe}{Ensemble Methods}
Different approaches:
\begin{itemize}
\item Model Averaging
\item Bagging
\item Boosting
\item Stacking
\end{itemize}
\vspace{1em}
Higher level mechanisms:
\begin{itemize}
    \begin{itemize}
        \item during base model training (how to generate different models?)
        \begin{itemize}
            \item input manipulation (e.g., bagging)
            \item partitioning of data
            \item learning algorithm manipulation (e.g., randomly select splits)
            \item output manipulation (error-correcting output codes)
            \item hybridization of the above
        \end{itemize}
        \item during prediction (how to combine the predictions of different models?)
        \begin{itemize}
            \item weighting methods
            \item meta-learning
        \end{itemize}
    \end{itemize}
\end{itemize}
\end{vbframe}

\begin{vbframe}{Model Averaging}

Definition: Combine predictions of multiple (different) models to make a final prediction\\
\begin{columns}
    \begin{column}{0.5\textwidth}
        \vspace{-1em}
        \begin{figure}
            \centering
            \includegraphics{slides/03-ensembles-stacking/figure/voting.png}
            %\caption{Caption}
            % from https://sebastianraschka.com/pdf/lecture-notes/stat479fs19/07-ensembles__notes.pdf
            \label{fig:voting}
        \end{figure}
    \end{column}
    \begin{column}{0.5\textwidth}
        Steps:
        \begin{enumerate}
            \item Base model training
            \item Combine predictions
        \end{enumerate}
    \end{column}
\end{columns}
\vspace{1em}
Combine predictions: majority voting (hard) / model averaging (soft)
\begin{itemize}
    \item $f^{[M]}(\mathbf{x}) = \text{mode}(\{f_{1}(\mathbf{x}), \ldots, f_{m}(\mathbf{x})\})$ assuming hard label predictions    
    \item $f^{[M]}(\mathbf{x}) = \frac{1}{M} \sum_{i = 1}^{M} f_{i}(\mathbf{x})$ assuming probability predictions
\end{itemize}
\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Weighted Model Averaging}

We can also introduce weights $\mathbf{w} = (w_{i}, \ldots, w_{m}), \text{usually: } 0 \le w_{i} \le 1, \sum_{i=1}^{M} w_{i} = 1$ accounting for the contribution of each individual model to the prediction of the ensemble: $f_{\mathbf{w}}^{[M]}(\mathbf{x}) = \frac{1}{M} \sum_{i = 1}^{M} w_{i} f_{i}(\mathbf{x})$\\
\vspace{1em}
This gives rise to an optimization problem to find the optimal weight vector so that the generalization error of the ensemble is optimal:
$\argmin_{\mathbf{w} \in \mathcal{W}} \hat{\text{GE}}(f_{\mathbf{w}}^{[M]})$\\

\end{vbframe}

\begin{vbframe}{Bagging}

Definition: Bagging (stands for Bootstrap Aggregating), training multiple instances of the same base model on different subsets of the training data, obtained through random sampling with replacement.\\
\begin{columns}
    \begin{column}{0.5\textwidth}
        \vspace{-1em}
        \begin{figure}
            \centering
            \includegraphics{slides/03-ensembles-stacking/figure/bagging.png}
            %\caption{Caption}
            % from https://sebastianraschka.com/pdf/lecture-notes/stat479fs19/07-ensembles__notes.pdf
            \label{fig:baggiing}
        \end{figure}
    \end{column}
    \begin{column}{0.5\textwidth}
        Steps:
        \begin{enumerate}
            \item Random sampling with replacement to generate different train data sets
            \item Base model training
            \item Combine predictions
        \end{enumerate}
    \end{column}
\end{columns}
\vspace{1em}
Popular use case: Construction of a random forest using decision trees as base models

\end{vbframe}

% ------------------------------------------------------------------------------

% ------------------------------------------------------------------------------

\begin{vbframe}{Bagging}

Why subsets of training data obtained through random sampling with replacement (input manipulation)
\begin{itemize}
    \item Introduce variability in base models
    \item Effective for cases where small changes in the training set may result in a completely different model (instable learners)
\end{itemize}
\vspace{1em}
Note that a random forest does not only employ input manipulation via bagging but also learning algorithm manipulation (sampling splits).

\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Boosting}
Definition: Boosting is an iterative technique that adjusts the weights of the training instances based on their misclassification rate or residual errors, and trains subsequent base models to focus on the misclassified or poorly predicted instances.

Steps:

\begin{itemize}
\item Initial model training:
\item Weight update
\item Base model training using updated weights
\item Repeat weight update
\item ...
\item Combine predictions
\end{itemize}
\end{vbframe}

% ------------------------------------------------------------------------------

\begin{vbframe}{Stacking}
Definition: Stacking (also stacked generalization or meta-ensemble), is an ensemble technique that involves training multiple base models on the same data and using their predictions as input features for a meta-model that makes the final ensemble prediction.

Steps:

\begin{itemize}
\item Base model training
\item Collect predictions
\item Meta model training
\end{itemize}

1 Level stacking vs. multi level stacking

In contrast to the other methods, stacking already combines model predictions during training

\end{vbframe}

% ------------------------------------------------------------------------------

Using cross-validated predictions of the base models in stacking improves generalization, reduces overfitting, enhances model diversity, and minimizes information leakage.
